# 模式C（买断 + 私域订阅）深度执行手册

版本：v1.0  
日期：2026-03-01  
适用项目：`Ai_auto_push`

## 1. 目标与结论

目标不是“只卖一次源码”，而是通过买断快速成交，再把客户分层转化到维护与托管，形成现金流 + 复利收入。

核心结论：

1. 闲鱼主卖买断，解决“获客”和“回款速度”。
2. 私域主卖维护/托管，解决“复购”和“收入稳定”。
3. 托管价不能一刀切低价，必须按“领域重叠率 + 服务级别”定价，否则容易陷入高运维低毛利。

## 2. 商业模型设计

### 2.1 三层产品梯度

| 层级 | 产品 | 定价建议 | 目标 |
|---|---|---:|---|
| 引流层 | 买断基础版 | ¥299（首发可限量¥199） | 降低首次决策门槛 |
| 利润层 | 买断标准/高级 | ¥599 / ¥999 | 拉高客单价与利润 |
| 复利层 | 月维护 / 托管订阅 | ¥99 / ¥199起 | 建立持续现金流 |

说明：托管不建议默认 ¥99/月。单客户独立运行成本在 ¥280-330 区间时，¥99 会稳定亏损。

### 2.2 客户分层与推荐路径

| 客户类型 | 特征 | 主推方案 |
|---|---|---|
| 动手型 | 能接受命令行，预算敏感 | 买断基础版 + 可选维护 |
| 结果型 | 不想折腾，只要结果 | 买断后升级托管订阅 |
| 生意型 | 关注行业机会、转化价值 | 高级版 + 托管 + 商机模块 |

推荐成交路径：

`闲鱼买断成交 -> 拉入微信私域 -> 7天内完成价值教育 -> 转化维护/托管`

## 3. 定价与毛利结构（模式C优化版）

### 3.1 对外报价卡

| 产品 | 价格 | 交付内容 | 毛利特点 |
|---|---:|---|---|
| 买断基础版 | ¥299 | 2领域、标准模板、部署教程 | 高毛利、快成交 |
| 买断标准版 | ¥599 | 4领域、品牌配色、多平台推送 | 客单价主力 |
| 买断高级版 | ¥999 | 6领域、定制UI、商机模块、1个月维护 | 利润最高 |
| 月维护 | ¥99/月 | 数据源修复、小功能更新 | 半被动收入 |
| 托管订阅（标准） | ¥199/月 | 4领域、托管运行、日报推送 | 需控成本 |
| 托管订阅（专业） | ¥299/月 | 6领域、优先支持、策略分析 | 适配高价值客户 |

### 3.2 托管订阅定价红线

当客户领域与已有客户重叠低（<40%）时，不建议低于 ¥199/月。  
当重叠高（>60%）时，可使用活动价 ¥149/月做限时转化。

## 4. 单位经济模型（按月）

### 4.1 关键假设

1. 买断平均客单价：¥450（¥299~¥999 混合后均值）。
2. 买断单均 API 成本：¥40。
3. 买断单均工时：3小时（按时间投入评估，不计外包人工现金成本）。
4. 托管平均 ARPU：¥220/月（标准/专业混合）。
5. 托管单客成本：¥90~¥180/月（取决于领域重叠与自动化复用）。

### 4.2 场景测算

| 场景 | 买断单量/月 | 买断收入 | 维护客户 | 托管客户 | 订阅收入 | 总收入 |
|---|---:|---:|---:|---:|---:|---:|
| 保守 | 4 | ¥1,800 | 2 | 1 | ¥418 | ¥2,218 |
| 基准 | 7 | ¥3,150 | 4 | 3 | ¥1,056 | ¥4,206 |
| 进取 | 10 | ¥4,500 | 6 | 5 | ¥1,694 | ¥6,194 |

备注：订阅收入按“维护¥99、托管均价¥220”估算。  
模式C的关键不是托管人数绝对值，而是“托管客户的重叠率与运营效率”。

## 5. 闲鱼到私域的漏斗设计

### 5.1 漏斗指标（建议监控）

| 阶段 | 指标目标（初期） | 指标目标（稳定期） |
|---|---:|---:|
| 曝光 -> 咨询 | 3% | 5% |
| 咨询 -> 成交 | 15% | 25% |
| 成交 -> 加微信 | 60% | 80% |
| 加微信 -> 维护订阅 | 15% | 25% |
| 加微信 -> 托管订阅 | 8% | 15% |

### 5.2 转化动作

1. 成交后 24 小时内发送“部署视频 + 常见报错 + 一键脚本”。
2. 第 3 天发送“自动更新失败自检清单”，自然引出维护服务。
3. 第 7 天发送“托管版对比表（省时成本）”，引导升级。
4. 第 14 天发送“本周行业摘要样例”，强化持续价值感知。

## 6. 交付与运营 SOP

### 6.1 买断交付 SOP

1. 需求确认：领域数量、推送平台、是否定制UI。
2. 环境确认：GitHub账号、部署方式、运行时间窗口。
3. 项目定制：配置领域、模板、推送渠道。
4. 演示验收：本地运行 + 线上访问 + 推送测试。
5. 交付归档：交付清单、教程链接、售后边界确认。

### 6.2 私域订阅 SOP

1. 客户建档：`客户ID/领域/版本/是否维护/是否托管/到期日`。
2. 托管分组：按领域做批处理分组，优先复用同源抓取结果。
3. 日常巡检：数据源可用性、任务成功率、推送到达率。
4. 异常处理：同类异常集中修复，避免逐个客户手工救火。
5. 月度续费：到期前 7 天、3 天、1 天自动提醒。

## 7. 自动化与技术落地建议

### 7.1 多客户配置结构（建议）

在项目内新增客户配置层：

1. `clients/{clientId}.json`：客户领域、推送渠道、品牌信息。
2. `profiles/{domain}.json`：领域抓取策略模板（可复用）。
3. 调度分为两类任务：`shared-fetch`（按领域） + `client-render-push`（按客户）。

这样可以把“重复抓取成本”降到最低，把算力主要花在客户个性化呈现和推送上。

### 7.2 最小SLA建议

| 指标 | 建议值 |
|---|---:|
| 日更成功率 | >= 95% |
| 推送到达率 | >= 98% |
| P1故障响应 | 2小时内 |
| 普通问题处理 | 24小时内 |

## 8. 风险控制与边界条款

### 8.1 必须写进服务说明

1. 数据源不可控：第三方网站改版/限流导致延迟属于外部风险。
2. 买断不含长期免费运维：免费支持仅限约定周期。
3. 托管服务边界：不包含客户新增大型功能开发。
4. 交付资产边界：源码交付后客户自行保管密钥与仓库权限。

### 8.2 防扯皮机制

1. 统一交付清单：功能点逐条勾选验收。
2. 统一故障分级：P1/P2/P3 响应与恢复时限明确。
3. 统一变更流程：新增需求走报价，不混入售后。

## 9. 90天执行计划

### 第1阶段（第1-2周）：打样与成交验证

1. 完成闲鱼商品页 3 个版本（基础/标准/高级）。
2. 打磨交付模板、部署视频、FAQ、报价单。
3. 获取首批 3-5 个买断客户，验证咨询->成交话术。

### 第2阶段（第3-6周）：私域转化与SOP固化

1. 搭建客户台账 + 到期提醒 + 售后工单表。
2. 跑通维护服务与托管服务的交付节奏。
3. 形成标准化转化节奏（D1/D3/D7/D14触达）。

### 第3阶段（第7-12周）：规模化与利润优化

1. 推行按领域分组调度，提升托管毛利。
2. 引入分层定价与年付折扣，提升现金流。
3. 每周复盘漏斗指标，迭代商品页与话术。

## 10. 核心看板（每周复盘）

1. 新增咨询数、成交数、成交率。
2. 买断客单价、退款率、交付时长。
3. 私域沉淀率（成交后加微信比例）。
4. 维护订阅数、托管订阅数、MRR。
5. 托管毛利率（按客户重叠率分层）。
6. 日更成功率、推送到达率、故障恢复时长。

## 11. 一句话策略

先用买断产品验证需求和现金流，再用维护与托管做复利；托管业务必须依赖“领域复用 + 自动化分组”才能长期健康盈利。
