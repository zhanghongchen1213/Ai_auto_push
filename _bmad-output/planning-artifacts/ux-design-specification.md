# UX Design Specification Ai_auto_push

**Author:** Xiaozhangxuezhang
**Date:** 2026-02-26

---

## Executive Summary

### Project Vision

Ai_auto_push 是一个面向个人使用的全自动化多领域智能资讯聚合平台。通过 Claude Code Skills 驱动 openclaw + 大模型，实现零人工干预的资讯抓取、AI 整理与自动发布闭环。核心体验承诺：每天早上打开一个页面，30秒内掌握 AI 技术、跨境电商、产品创业、GitHub 热门项目四大领域的关键动态，感兴趣的内容一键直达原文。

产品定位为"精选而非轰炸"的个人信息基础设施，通过结构化每日摘要替代碎片化的多平台信息获取方式。

### Target Users

**主要用户：Xiaozhangxuezhang**
- 多领域关注者，同时跟踪 AI、跨境电商、产品创业、GitHub 等方向
- 核心场景：每日晨间办公桌前的信息速览（PC 为主）
- 次要场景：移动端碎片时间浏览、历史资讯回溯
- 技术水平：中级，能通过 Skills 配置扩展领域，无需写代码
- 当前阶段为纯个人工具，单用户场景

**用户心智模型：**
- 期望"打开即看"的零操作成本体验
- 信任 AI 整理的摘要质量，但需要原文链接作为验证出口
- 对信息时效性敏感（资讯必须是当天的）
- 习惯扫描式阅读，先看标题再决定是否深入

### Key Design Challenges

1. **信息密度与可扫描性平衡**：4个领域×多条资讯，需在30秒浏览窗口内实现高效信息传递，避免认知过载
2. **时间维度导航**：今日资讯与历史回溯的无缝切换，时间线导航需直觉化且不干扰主阅读流
3. **跨端响应式体验**：桌面多列→移动单列的布局转换需保持浏览效率，而非简单堆叠
4. **领域视觉区分**：四大领域需要清晰的视觉边界，同时保持页面整体的视觉统一性
5. **搜索与浏览的模式切换**：从时间线浏览到关键词搜索的交互切换需流畅自然

### Design Opportunities

1. **晨报式阅读体验**：借鉴纸质晨报的信息层级设计，用视觉权重引导注意力流向
2. **领域色彩编码系统**：为四大领域建立独特视觉标识，形成用户肌肉记忆
3. **渐进式信息展开**：标题→摘要→原文的三层信息架构，让用户按需控制信息消费深度
4. **时间感知设计**：通过视觉线索强化"今天"的时间锚点，让历史导航有清晰的时间方位感

## Core User Experience

### Defining Experience

核心体验是"扫描式速览"——用户打开页面后，通过自上而下的视觉扫描完成信息获取。核心交互循环为：

**扫描 → 发现 → 深入**

1. **扫描**：眼睛快速掠过各领域标题，建立今日信息全景
2. **发现**：在某个标题处停下，阅读150-200字摘要确认价值
3. **深入**：点击原文链接跳转至信息源，获取完整内容

这个循环应在30秒内完成一轮完整扫描，用户可按需在任意环节停下深入。整个体验类似翻阅一份为个人定制的每日晨报。

### Platform Strategy

- **主平台**：Web（SSG 静态站点），桌面端浏览器为核心场景
- **次要平台**：移动端响应式适配（碎片时间浏览）
- **交互模式**：鼠标/键盘为主，触屏为辅
- **离线需求**：无（纯在线消费型产品）
- **性能约束**：FCP≤1.5s / LCP≤2.5s / 页面≤500KB
- **部署模式**：CDN 托管静态资源，Git push 触发自动构建

### Effortless Interactions

以下交互必须做到零摩擦：

1. **打开即看**：页面加载完成即展示今日资讯，无需任何用户操作（无登录、无选择、无加载等待）
2. **领域一眼识别**：通过色彩编码和视觉分区，用户无需阅读标签即可识别内容所属领域
3. **时间一键切换**：从今天到昨天、从本周到上周，每次时间跳转只需一次点击
4. **搜索随时可达**：搜索入口常驻可见，输入即搜，结果即时呈现（≤500ms）
5. **原文无缝跳转**：点击链接在新标签页打开原文，不中断当前浏览上下文

### Critical Success Moments

| 时刻 | 用户感受 | 设计要求 |
|------|----------|----------|
| 首次打开 | "信息已经准备好了" | 零配置、零等待的首屏体验 |
| 意外发现 | "这个我差点错过" | 跨领域聚合的独特价值体现 |
| 效率对比 | "比刷公众号快多了" | 30秒完成扫描的信息密度设计 |
| 历史查找 | "2分钟就找到了" | 时间线+搜索的双路径检索 |
| 领域扩展 | "明天就能看到新领域" | Skills 配置的即时生效感知 |

### Experience Principles

1. **零操作成本**：用户打开页面的那一刻，价值已经呈现。不要求用户做任何事情就能获得核心体验。
2. **信息层级清晰**：标题→摘要→原文，三层渐进式展开。每一层都是完整的信息单元，用户可在任意层级停下。
3. **时间是第一维度**：资讯以时间为主轴组织，"今天"永远是默认视图，历史是可选的深入路径。
4. **领域是第二维度**：在时间维度内，领域分区提供横向的信息组织，让用户按兴趣选择性深入。
5. **尊重注意力**：精选而非轰炸，每条资讯都值得出现。页面不追求信息量最大化，而追求信息价值密度最大化。

## Desired Emotional Response

### Primary Emotional Goals

1. **掌控感（In Control）**：用户感到自己掌握了信息全局，不再被碎片化渠道牵着走。"我知道今天发生了什么"的确定感。
2. **高效感（Efficient）**：5分钟完成以前30分钟的事，时间被尊重的满足感。"这才是信息获取该有的样子"。
3. **发现的惊喜（Serendipity）**：跨领域聚合带来意外发现——"如果不是这个平台，我可能就错过了"。
4. **平静专注（Calm Focus）**：不是信息流的焦虑滚动，而是晨报式的从容浏览。信息为我服务，而非我被信息淹没。

### Emotional Journey Mapping

| 阶段 | 期望情感 | 设计支撑 |
|------|----------|----------|
| 首次访问 | 好奇 → 惊喜："原来可以这样获取信息" | 零配置首屏即展示完整资讯，无需引导流程 |
| 每日速览 | 从容 → 掌控："今天的信息我已掌握" | 清晰的领域分区 + 可扫描的标题层级 |
| 发现有价值内容 | 兴奋 → 满足："这个太有用了" | 摘要提供足够判断依据，链接一键直达 |
| 历史回溯 | 确定 → 高效："很快就找到了" | 时间线直觉导航 + 搜索双路径 |
| 遇到问题 | 困惑 → 安心："知道怎么排查" | Git 日志可观测，错误信息清晰 |
| 再次回来 | 期待 → 习惯："每天第一件事就是打开它" | 一致的体验模式，形成肌肉记忆 |

### Micro-Emotions

**需要强化的正向微情感：**
- **信任**：AI 整理的摘要准确可靠，原文链接始终可用
- **确定**：信息是今天的、是完整的、没有遗漏
- **轻松**：浏览过程无认知负担，信息层级自然引导注意力
- **成就**：快速完成信息获取，带着清晰视野开始工作

**需要避免的负向微情感：**
- **焦虑**：信息过多导致的"还没看完"压力
- **怀疑**：摘要质量不稳定导致的信任缺失
- **迷失**：在历史日期中找不到方向
- **失望**：某个领域今天没有内容更新

### Design Implications

| 情感目标 | UX 设计策略 |
|----------|-------------|
| 掌控感 | 信息全景一屏可见，领域分区清晰，无需翻页即可建立全局认知 |
| 高效感 | 标题字号突出便于扫描，摘要长度统一（150-200字），视觉节奏一致 |
| 发现惊喜 | 跨领域内容并列展示，不同领域的视觉差异化吸引注意力流动 |
| 平静专注 | 大量留白、克制的色彩使用、无动画干扰、无弹窗打断 |
| 信任 | 每条资讯标注来源和日期，原文链接醒目可见 |
| 确定 | 页面顶部明确显示"今日日期"和"更新状态" |

### Emotional Design Principles

1. **宁静优先**：界面设计追求报纸般的沉稳感，避免社交媒体式的刺激和焦虑。用留白和排版节奏创造阅读的从容感。
2. **确定性设计**：每个界面元素都传递明确信息——今天的日期、更新状态、内容来源。消除一切模糊和不确定。
3. **渐进式满足**：从标题的快速扫描到摘要的深入了解再到原文的完整阅读，每一步都提供即时的信息满足感。
4. **一致性即信任**：每天相同的布局、相同的位置、相同的格式。一致性建立习惯，习惯建立信任。

## UX Pattern Analysis & Inspiration

### Inspiring Products Analysis

**1. ai-bot.cn/daily-ai-news（PRD 明确参考）**
- 时间轴纵向排列，每日一页，结构清晰
- 标题+摘要+链接的三层信息架构，扫描效率高
- 单领域聚焦，信息密度适中
- 可借鉴：时间轴布局模式、资讯卡片结构
- 需超越：仅单领域、无搜索、无响应式适配

**2. Hacker News**
- 极简列表式布局，信息密度极高
- 标题即入口，一行一条，扫描速度极快
- 无摘要层——点击即跳转原文
- 可借鉴：极简主义设计哲学、高信息密度的列表布局
- 需超越：无领域分区、无摘要、视觉层级单一

**3. Morning Brew（邮件晨报）**
- "晨报"概念的最佳实践——每日定时、精选内容、结构化呈现
- 分区明确（科技/商业/市场），色彩编码区分领域
- 摘要精炼，语言轻松，阅读体验愉悦
- 可借鉴：晨报式信息架构、领域色彩编码、精炼摘要风格
- 需超越：邮件形式无法搜索历史、无时间线导航

### Transferable UX Patterns

**导航模式：**
- 时间轴纵向导航（ai-bot.cn）→ 用于每日资讯的主浏览模式
- 日期选择器快速跳转 → 用于历史回溯场景
- 顶部常驻搜索栏（Hacker News）→ 用于关键词检索

**交互模式：**
- 卡片式资讯展示（标题+摘要+链接）→ 核心内容单元
- 领域标签/色彩编码（Morning Brew）→ 快速领域识别
- 外部链接新标签页打开 → 不中断浏览上下文

**视觉模式：**
- 大量留白 + 清晰排版层级 → 支撑"平静专注"情感目标
- 领域专属色彩系统 → 建立视觉肌肉记忆
- 统一的卡片/条目高度 → 创造视觉节奏感

### Anti-Patterns to Avoid

1. **无限滚动信息流**：社交媒体式的无尽滚动制造焦虑，与"精选而非轰炸"理念冲突。每日内容应有明确边界。
2. **过度动画/转场**：资讯阅读场景追求效率，花哨动画只会拖慢浏览速度。
3. **弹窗/横幅干扰**：任何打断阅读流的元素（订阅弹窗、Cookie 提示等）都会破坏"平静专注"体验。
4. **隐藏式导航**：汉堡菜单等隐藏导航增加操作成本，关键导航（日期、搜索、领域）应始终可见。
5. **信息过载的仪表盘**：避免把页面做成数据仪表盘，保持晨报的简洁叙事感。

### Design Inspiration Strategy

**采纳（Adopt）：**
- ai-bot.cn 的时间轴纵向布局——已验证的资讯浏览模式
- Morning Brew 的领域色彩编码——快速建立领域识别
- Hacker News 的极简主义哲学——内容优先，装饰最少

**适配（Adapt）：**
- ai-bot.cn 的单领域布局 → 扩展为多领域分区并列
- Morning Brew 的邮件格式 → 转化为可搜索的 Web 页面
- Hacker News 的纯标题列表 → 增加摘要层提供更多判断依据

**规避（Avoid）：**
- 社交媒体的无限滚动和算法推荐
- 新闻门户的广告密集布局
- SaaS 产品的复杂仪表盘风格

## Design System Foundation

### Design System Choice

**方案3：可主题化系统（Themeable System）— Tailwind CSS**

选择 Tailwind CSS 作为 Ai_auto_push 的设计系统基础。这是一个实用优先（utility-first）的 CSS 框架，提供高度可定制的设计令牌系统，同时保持极小的生产构建体积。

### Rationale for Selection

1. **与 Astro SSG 完美集成**：Tailwind 原生支持 Astro，构建时自动 tree-shaking 未使用的样式，确保页面 ≤500KB 的性能约束
2. **单人开发效率最大化**：utility-first 模式无需在 HTML 和 CSS 文件间切换，开发速度显著提升
3. **设计令牌系统**：通过 `tailwind.config` 定义色彩、间距、字体等设计令牌，确保四大领域的视觉一致性
4. **响应式设计内建**：`sm:` / `md:` / `lg:` 断点前缀天然支持桌面→移动端的响应式适配需求
5. **极小的学习曲线**：相比 Material UI 或 Ant Design 的组件 API，Tailwind 的 utility class 更直觉化
6. **无运行时开销**：纯 CSS 输出，零 JavaScript 运行时，完美契合 SSG 静态站点的性能要求

### Implementation Approach

- **Tailwind CSS v4**：使用最新版本，利用 CSS-first 配置和原生 CSS 变量
- **Astro 集成**：通过 `@astrojs/tailwind` 官方集成，零配置启用
- **设计令牌定义**：在 `tailwind.config` 中集中管理色彩、字体、间距、断点等令牌
- **组件级样式**：在 Astro 组件中直接使用 utility class，无需额外 CSS 文件
- **暗色模式预留**：通过 `dark:` 前缀预留暗色模式扩展能力（当前阶段不实现）

### Customization Strategy

**领域色彩令牌：**
- AI 技术领域：蓝色系（`--color-ai`）
- 跨境电商领域：绿色系（`--color-ecommerce`）
- 产品创业领域：橙色系（`--color-startup`）
- GitHub 热门领域：紫色系（`--color-github`）

**排版令牌：**
- 标题字体：系统无衬线字体栈（-apple-system, system-ui）
- 正文字体：同上，确保跨平台一致性
- 标题层级：h1(24px) → h2(20px) → h3(16px)，扫描友好的字号梯度
- 摘要正文：14px，行高1.6，优化阅读舒适度

**间距系统：**
- 基础单位：4px
- 卡片内边距：16px（移动端）/ 24px（桌面端）
- 领域分区间距：32px
- 页面边距：16px（移动端）/ 48px（桌面端）/ 64px（宽屏）

## Defining Core Experience

### Defining Experience Statement

**"打开即览，30秒掌握四大领域今日动态"**

如同 Tinder 的"滑动匹配"、Spotify 的"即时播放任何歌曲"，Ai_auto_push 的定义体验是：用户打开页面的瞬间，一份为个人定制的多领域智能晨报已经准备就绪——无需登录、无需配置、无需等待。眼睛自上而下扫过四个色彩分区，30秒内建立今日信息全景。

### User Mental Model

**当前解决方案与痛点：**
- 用户目前通过多个渠道获取信息：微信公众号、Twitter/X、Hacker News、各类 Newsletter
- 每天需要在5-8个平台间切换，耗时30-60分钟
- 信息碎片化严重，容易遗漏重要动态
- 无法快速建立跨领域的信息全景

**用户心智模型：**
- 将 Ai_auto_push 理解为"个人定制的每日晨报"——类似纸质报纸的数字化升级
- 期望"打开即看"的零操作体验，如同打开报纸翻阅头版
- 信任 AI 的筛选和整理能力，但需要原文链接作为"事实核查"出口
- 时间维度是第一直觉——"今天发生了什么"是最自然的信息组织方式

### Success Criteria

| 指标 | 成功标准 | 衡量方式 |
|------|----------|----------|
| 首屏加载 | ≤1.5s FCP | Lighthouse 性能审计 |
| 扫描完成 | 30秒内完成一轮全领域扫描 | 页面信息密度设计验证 |
| 信息完整 | 四大领域每日均有内容更新 | 自动化流水线监控 |
| 原文可达 | 100% 链接可点击跳转 | 链接有效性检查 |
| 历史可查 | 2分钟内找到任意历史资讯 | 搜索+时间线双路径验证 |

### Novel UX Patterns

**模式类型：已验证模式的创新组合**

本项目不需要发明全新的交互模式，而是将多个已验证的 UX 模式进行创新组合：

- **时间轴浏览**（已验证：ai-bot.cn）+ **多领域色彩分区**（已验证：Morning Brew）= 多领域时间轴晨报
- **卡片式信息展示**（已验证：通用模式）+ **三层渐进展开**（标题→摘要→原文）= 可控深度的信息消费
- **静态站点生成**（已验证：技术模式）+ **Pagefind 客户端搜索**（已验证）= 零服务器的全文检索

**独特之处：**
- 四大领域并列展示的"信息全景"视图——市面上没有同类产品提供这种跨领域聚合体验
- 完全自动化的内容生产流水线——用户看到的是结果，背后是 AI 驱动的零人工干预

### Experience Mechanics

**1. 启动（Initiation）：**
- 用户打开浏览器书签或输入网址
- 页面在1.5秒内完成首屏渲染
- 今日日期和更新状态立即可见于页面顶部

**2. 扫描（Interaction）：**
- 四大领域以色彩分区纵向排列
- 每个领域展示3-5条精选资讯标题
- 用户眼睛自上而下扫描，色彩变化引导注意力在领域间流动

**3. 反馈（Feedback）：**
- 领域色彩编码提供即时的位置感知
- 标题字号和粗细传递信息重要性层级
- 摘要区域的展开/收起提供交互确认

**4. 完成（Completion）：**
- 页面有明确的内容边界（非无限滚动）
- 扫描到底部即意味着"今日资讯已全部浏览"
- 时间线导航提供"还有昨天的"延伸路径

### Accessibility Considerations

**色彩对比度：**

## Design Direction Decision

### Design Directions Explored

通过 Pencil 设计工具生成了完整的桌面端首页 Mockup，探索了以下设计方向：

**方向1：资讯列表式布局（News List Feed）— 最终选定**
- 参考 ai-bot.cn 的资讯聚合风格，采用更现代的卡片式设计
- 单列纵向信息流，顶部水平 pill 标签导航实现领域切换
- 领域通过彩色 pill 标签区分，而非独立分区
- 日期分组头 + 白色卡片列表，信息流统一，扫描效率高
- 整体风格：现代简洁，信息密度适中，领域切换便捷

**方向2：编辑式晨报布局（Editorial Morning Briefing）**
- 单列纵向信息流，四大领域依次排列
- 每个领域通过色彩编码（蓝/绿/橙/紫）建立视觉边界
- 卡片式资讯展示：标题→摘要→来源+链接
- 整体风格：大量留白、克制色彩、报纸般的沉稳感

**其他探索方向（未选定）：**
- 方向3：仪表盘式网格布局 — 四领域并列2×2网格，信息密度高但扫描路径不够线性
- 方向4：标签页切换布局 — 每个领域独立标签页，减少信息过载但失去跨领域全景感
- 方向5：时间线混合流 — 所有领域按时间混排，发现感强但领域边界模糊

### Chosen Direction

**方案一：资讯列表式布局（News List Feed）**

选择单列纵向信息流布局作为核心设计方向，参考 ai-bot.cn 的资讯聚合风格，但采用更现代的卡片式设计。顶部水平 pill 标签导航实现领域快速切换（全部/AI技术/跨境电商/产品创业/GitHub热门），内容区以日期分组头 + 白色新闻卡片列表呈现，领域通过彩色 pill 标签区分而非独立分区。搜索功能常驻于 Header 右侧，随时可达。

### Design Rationale

1. **信息流统一，扫描效率高**：单列纵向信息流避免了多列布局的视线跳转，用户自上而下线性扫描即可完成全部浏览，符合"30秒速览"的核心体验目标
2. **领域切换便捷**：顶部水平 pill 标签导航支持一键切换领域筛选，彩色标签提供即时的领域视觉识别，无需滚动查找
3. **搜索功能常驻**：搜索框固定在 Header 右侧，随时可达，支持关键词检索历史资讯
4. **现代卡片式设计**：白色卡片 + 浅灰背景的层次感，配合圆角和适当间距，视觉清爽现代
5. **响应式友好**：单列布局天然适配移动端，无需复杂的多列折叠策略，仅需调整间距和字号即可完成跨端适配

### Implementation Approach

**页面结构：**
- Header：白色背景(#FFFFFF)，高54px，水平排列，包含品牌Logo("📡 资讯雷达")、pill标签导航、搜索框
- 内容区：#F5F7FA背景，垂直排列，gap 24，padding 24 48，包含日期分组头 + 新闻卡片列表
- Footer：极简底部，仅包含版权和 GitHub 链接

**组件复用策略：**
- `NewsCard` 组件：白色卡片，圆角8px，包含领域pill标签 + 来源时间 + 标题 + 摘要 + 阅读原文链接
- `CategoryNav` 组件：水平pill标签导航，支持领域筛选切换，各领域独立配色
- `DateGroupHeader` 组件：蓝色竖条 + 日期文字 + 计数badge + 更新时间
- `SearchBar` 组件：圆角20px搜索框，集成 Pagefind 客户端搜索
- `DateNav` 组件：时间线导航，支持日期快速跳转

**Pencil 设计文件：** `pencil-new.pen` 包含完整桌面端首页 Mockup，供 dev-story 阶段读取参考

## Performance Targets

### 核心 Web 指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| FCP（首次内容绘制） | ≤ 1.5s | 用户感知到页面开始加载的时间 |
| LCP（最大内容绘制） | ≤ 2.5s | 主要内容区域完成渲染的时间 |
| 首屏页面总大小 | ≤ 500KB | 包含 HTML、CSS、JS、字体等所有资源 |

### 加载与渲染策略

- **静态站点预渲染**：通过 Astro SSG 在构建时生成全部 HTML 页面，零服务端运行时开销
- **CDN 分发**：静态资源通过 CDN 边缘节点分发，缩短用户端加载延迟
- **新闻列表分页加载**：每页加载 20 条资讯，避免一次性渲染大量 DOM 节点；当资讯总量较大时可考虑虚拟滚动方案
- **图片懒加载**：如资讯包含缩略图，使用 `loading="lazy"` 延迟加载视口外图片
- **CSS Tree-shaking**：Tailwind CSS 构建时自动移除未使用的样式类，确保 CSS 体积最小化
